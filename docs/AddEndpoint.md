# AddEndpoint.md — Guía para agregar un módulo o endpoint

Guía completa para agregar funcionalidad nueva al proyecto desde cero. Incluye flujo por capas, patrones de DI, variantes de respuesta y checklist pre-build.

---

## Visión general

Agregar un endpoint siempre implica tocar **todas las capas** en orden. El orden importa porque las capas superiores dependen de las inferiores:

```
1. SQL migration          (Host)
2. Domain entity          (Domain)
3. Repository interface   (Domain)
4. ...Sql class           (Infrastructure)
5. Repository impl        (Infrastructure)
6. DI Infrastructure      (Infrastructure/ServiceCollectionEx.cs)
7. DTO                    (Application)
8. Use case               (Application)  ← Request + Handler + Responses/
9. DI Application         (automático — AddMediator escanea el ensamblado)
10. Presenter             (WebApi)
11. Controller / endpoint (WebApi)
12. DI WebApi             (WebApi/ServiceCollectionEx.cs)
```

---

## Ejemplo completo: módulo `Products`

Vamos a agregar un módulo de productos con un endpoint `GET /api/products/{id}`.

---

### Paso 1 — Migración SQL

`Host/Services/Schema Migration/Tables/010_products.sql`

```sql
CREATE TABLE IF NOT EXISTS dbo.Products
(
    Id           BIGINT       GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    PublicId     UUID         NOT NULL DEFAULT gen_random_uuid(),
    Name         VARCHAR(200) NOT NULL,
    Description  TEXT         NOT NULL DEFAULT '',
    Price        NUMERIC(12,2) NOT NULL DEFAULT 0,
    IsActive     BOOLEAN      NOT NULL DEFAULT TRUE,
    CreatedAtUtc TIMESTAMP(0) NOT NULL DEFAULT (timezone('utc', now())),
    UpdatedAtUtc TIMESTAMP(0) NOT NULL DEFAULT (timezone('utc', now())),

    CONSTRAINT UQ_Products_Name UNIQUE (Name),
    CONSTRAINT CK_Products_Price CHECK (Price >= 0)
);
```

`Host/Services/Schema Migration/Tables/011_products_indexes.sql`

```sql
CREATE UNIQUE INDEX IF NOT EXISTS UX_Products_PublicId ON dbo.Products (PublicId);
CREATE INDEX IF NOT EXISTS IX_Products_IsActive    ON dbo.Products (IsActive);
CREATE INDEX IF NOT EXISTS IX_Products_CreatedAt   ON dbo.Products (CreatedAtUtc DESC);
CREATE INDEX IF NOT EXISTS IX_Products_Name        ON dbo.Products (LOWER(Name));
```

> Las migraciones se ejecutan **automáticamente** al iniciar la aplicación. Nunca ejecutar SQL manualmente contra la base de datos. Ver [DB.md](DB.md) para la referencia completa.

### Paso 2 — Entidad de dominio

`Domain/Entities/Products/Product.cs`

```csharp
namespace Domain.Entities.Products;

public sealed class Product
{
    public long     Id           { get; init; }
    public Guid     PublicId     { get; init; }
    public string   Name         { get; init; } = string.Empty;
    public string   Description  { get; init; } = string.Empty;
    public decimal  Price        { get; init; }
    public bool     IsActive     { get; init; }
    public DateTime CreatedAtUtc { get; init; }
    public DateTime UpdatedAtUtc { get; init; }
}
```

**Reglas de las entidades de dominio:**
- Propiedades `init` — inmutables después de crear
- Sin lógica de negocio (es un record de datos)
- Sin referencias a capas externas
- Mapea 1:1 con las columnas de la tabla

### Paso 3 — Interfaz de repositorio

`Domain/Repositories/Products/IProductRepository.cs`

```csharp
using Domain.Entities.Products;

namespace Domain.Repositories.Products;

public interface IProductRepository
{
    Task<Product?> GetByPublicIdAsync(Guid publicId, CancellationToken cancellationToken = default);
    Task<bool> ExistsByNameAsync(string name, CancellationToken cancellationToken = default);
    Task<IReadOnlyCollection<Product>> GetPagedAsync(int page, int pageSize, CancellationToken cancellationToken = default);
    Task<int> GetCountAsync(CancellationToken cancellationToken = default);
    Task<Product> InsertAsync(string name, string description, decimal price, CancellationToken cancellationToken = default);
    Task<int> UpdateAsync(Guid publicId, string name, string description, decimal price, CancellationToken cancellationToken = default);
    Task<int> DisableAsync(Guid publicId, CancellationToken cancellationToken = default);
}
```

**Reglas de las interfaces de repositorio:**
- Viven en `Domain/` — sin referencias a Infrastructure
- Retornan entidades de dominio, nunca tipos de Dapper ni DTOs
- Métodos async con `CancellationToken`
- La conversión de `IEnumerable<T>` → `IReadOnlyCollection<T>` ocurre en la implementación

### Paso 4 — Clase Sql

`Infrastructure/Persistence/SQLDB/Main/Products/ProductsSql.cs`

```csharp
using Domain.Entities.Products;
using Infrastructure.PostgreSql;

namespace Infrastructure.Persistence.SQLDB.Main.Products;

public sealed class ProductsSql
{
    private readonly MainDapperDbConnection _db;

    public ProductsSql(MainDapperDbConnection db) => _db = db;

    public Task<Product?> GetByPublicIdAsync(Guid publicId, CancellationToken ct = default) =>
        _db.QuerySingleAsync<Product>(
            """
            SELECT Id, PublicId, Name, Description, Price, IsActive, CreatedAtUtc, UpdatedAtUtc
            FROM dbo.Products
            WHERE PublicId = @publicId;
            """,
            new { publicId },
            cancellationToken: ct);

    public Task<bool> ExistsByNameAsync(string name, CancellationToken ct = default) =>
        _db.ExecuteScalarAsync<bool>(
            """
            SELECT EXISTS (
                SELECT 1 FROM dbo.Products
                WHERE LOWER(Name) = LOWER(@name)
            );
            """,
            new { name },
            cancellationToken: ct);

    public Task<IEnumerable<Product>> GetPagedAsync(int page, int pageSize, CancellationToken ct = default) =>
        _db.QueryAsync<Product>(
            """
            SELECT Id, PublicId, Name, Description, Price, IsActive, CreatedAtUtc, UpdatedAtUtc
            FROM dbo.Products
            WHERE IsActive = TRUE
            ORDER BY CreatedAtUtc DESC
            LIMIT @pageSize OFFSET @offset;
            """,
            new { offset = (page - 1) * pageSize, pageSize },
            cancellationToken: ct);

    public Task<int> GetCountAsync(CancellationToken ct = default) =>
        _db.ExecuteScalarAsync<int>(
            """
            SELECT COUNT(*)::int FROM dbo.Products WHERE IsActive = TRUE;
            """,
            cancellationToken: ct);

    public async Task<Product> InsertAsync(
        string name, string description, decimal price,
        CancellationToken ct = default) =>
        (await _db.QueryFirstAsync<Product>(
            """
            INSERT INTO dbo.Products (Name, Description, Price)
            VALUES (@name, @description, @price)
            RETURNING Id, PublicId, Name, Description, Price, IsActive, CreatedAtUtc, UpdatedAtUtc;
            """,
            new { name, description, price },
            cancellationToken: ct).ConfigureAwait(false))!;

    public Task<int> UpdateAsync(
        Guid publicId, string name, string description, decimal price,
        CancellationToken ct = default) =>
        _db.ExecuteAsync(
            """
            UPDATE dbo.Products
            SET Name         = @name,
                Description  = @description,
                Price        = @price,
                UpdatedAtUtc = timezone('utc', now())
            WHERE PublicId = @publicId
              AND IsActive  = TRUE;
            """,
            new { publicId, name, description, price },
            cancellationToken: ct);

    public Task<int> DisableAsync(Guid publicId, CancellationToken ct = default) =>
        _db.ExecuteAsync(
            """
            UPDATE dbo.Products
            SET IsActive     = FALSE,
                UpdatedAtUtc = timezone('utc', now())
            WHERE PublicId = @publicId
              AND IsActive  = TRUE;
            """,
            new { publicId },
            cancellationToken: ct);
}
```

**Reglas de las clases Sql:**
- Una clase por tabla
- Inyectan `MainDapperDbConnection` por constructor — nunca Dapper directamente
- SQL como raw strings `"""..."""` — nunca concatenación ni interpolación
- Parámetros siempre como objeto anónimo `new { param }`
- Retornan entidades de dominio directamente — sin Row classes intermedias
- Para `InsertAsync` con `RETURNING`: usar `QueryFirstAsync` con `!` null-forgiving

### Paso 5 — Implementación del repositorio

`Infrastructure/Repositories/Products/ProductRepository.cs`

```csharp
using Domain.Entities.Products;
using Domain.Repositories.Products;
using Infrastructure.Persistence.SQLDB.Main.Products;

namespace Infrastructure.Repositories.Products;

public sealed class ProductRepository : IProductRepository
{
    private readonly ProductsSql _sql;

    public ProductRepository(ProductsSql sql) => _sql = sql;

    public Task<Product?> GetByPublicIdAsync(Guid publicId, CancellationToken cancellationToken = default) =>
        _sql.GetByPublicIdAsync(publicId, cancellationToken);

    public Task<bool> ExistsByNameAsync(string name, CancellationToken cancellationToken = default) =>
        _sql.ExistsByNameAsync(name, cancellationToken);

    public async Task<IReadOnlyCollection<Product>> GetPagedAsync(int page, int pageSize, CancellationToken cancellationToken = default) =>
        (await _sql.GetPagedAsync(page, pageSize, cancellationToken)).ToList().AsReadOnly();

    public Task<int> GetCountAsync(CancellationToken cancellationToken = default) =>
        _sql.GetCountAsync(cancellationToken);

    public Task<Product> InsertAsync(string name, string description, decimal price, CancellationToken cancellationToken = default) =>
        _sql.InsertAsync(name, description, price, cancellationToken);

    public Task<int> UpdateAsync(Guid publicId, string name, string description, decimal price, CancellationToken cancellationToken = default) =>
        _sql.UpdateAsync(publicId, name, description, price, cancellationToken);

    public Task<int> DisableAsync(Guid publicId, CancellationToken cancellationToken = default) =>
        _sql.DisableAsync(publicId, cancellationToken);
}
```

#### `Application/UseCases/Products/GetProduct/GetProductHandler.cs`

```csharp
using Application.Dto.Products;
using Common.Messaging;
using Domain.Repositories.Products;

namespace Application.UseCases.Products.GetProduct;

public sealed class GetProductHandler : IRequestHandler<GetProductRequest, GetProductResponse>
{
    private readonly IProductRepository _repo;

    public GetProductHandler(IProductRepository repo) => _repo = repo;

    public async Task<GetProductResponse> Handle(
        GetProductRequest request, CancellationToken cancellationToken)
    {
        var product = await _repo.GetByPublicIdAsync(request.PublicId, cancellationToken);
        if (product is null)
            return new GetProductNotFoundFailure("Producto no encontrado.");

        return new GetProductSuccess(new ProductDto(
            product.PublicId, product.Name, product.Description,
            product.Price, product.IsActive, product.CreatedAtUtc, product.UpdatedAtUtc));
    }
}
```

> **Nota DI Application:** Los handlers son descubiertos y registrados **automáticamente** por `AddMediator(Assembly.GetExecutingAssembly())` al iniciar. No es necesario registrarlos manualmente.

### Paso 9 — Presenter

`WebApi/EndPoints/Products/Presenters/GetProductPresenter.cs`

```csharp
using Application.Dto.Products;
using Application.UseCases.Products.GetProduct;
using Common.Abstractions;
using Common.Results;
using Common.ViewModels;

namespace WebApi.EndPoints.Products.Presenters;

public sealed class GetProductPresenter : IPresenter<GetProductResponse>
{
    private readonly ResultViewModel<ProductsController> _viewModel;

    public GetProductPresenter(ResultViewModel<ProductsController> viewModel)
        => _viewModel = viewModel;

    public Task Handle(GetProductResponse notification, CancellationToken cancellationToken)
    {
        if (notification is IFailure failure)
            _viewModel.Fail(failure.Message);
        else if (notification is ISuccess<ProductDto> success)
            _viewModel.Set(success);

        return Task.CompletedTask;
    }
}
```

### Paso 10 — Controller

`WebApi/EndPoints/Products/ProductsController.cs`

```csharp
using Application.UseCases.Products.GetProduct;
using Common.Messaging;
using Common.ViewModels;
using Microsoft.AspNetCore.Mvc;
using Microsoft.Extensions.Logging;
using WebApi.Base;

namespace WebApi.EndPoints.Products;

[Route("api/products")]
// [Authorize]   ← descomentar cuando se requiera autenticación
public sealed class ProductsController : BaseApiController
{
    private readonly ILogger<ProductsController> _logger;
    private readonly ResultViewModel<ProductsController> _viewModel;

    public ProductsController(
        IMediator mediator,
        ILogger<ProductsController> logger,
        ResultViewModel<ProductsController> viewModel) : base(mediator)
    {
        _logger    = logger;
        _viewModel = viewModel;
    }

    [HttpGet("{id:guid}")]
    public async Task<IActionResult> GetById(Guid id, CancellationToken ct = default)
    {
        try
        {
            _ = await Mediator.Send(new GetProductRequest(id), ct);
            return _viewModel.IsSuccess ? Ok(_viewModel) : StatusCode(500, _viewModel);
        }
        catch (Exception ex)
        {
            _logger.LogError(ex, "Error en GetById Product");
            var innerEx = ex;
            while (innerEx.InnerException != null) innerEx = innerEx.InnerException!;
            return StatusCode(500, _viewModel.Fail(innerEx.Message));
        }
    }
}
```

## Variantes de respuesta del Presenter

Según lo que retorne el use case, el presenter usa un método diferente de `ResultViewModel<T>`:

### Variante A — Éxito con dato único (`ISuccess<TDto>`)

```csharp
// GetProductSuccess(ProductDto Data) : ..., ISuccess<ProductDto>

else if (notification is ISuccess<ProductDto> success)
    _viewModel.Set(success);   // Data = success.Data (el DTO)
```

Usar cuando el resultado es un objeto único con forma bien definida.

### Variante B — Éxito con datos custom (paginación, colecciones)

```csharp
// GetProductsSuccess(IReadOnlyCollection<ProductDto> Items, int Total, int Page, int PageSize) : ..., ISuccess

else if (notification is GetProductsSuccess success)
    _viewModel.OK(success);    // Data = el record completo con todos sus campos
```

Usar cuando el resultado tiene múltiples campos que no caben en `ISuccess<T>` sin crear referencia circular.

### Variante C — Éxito sin datos (update, disable)

```csharp
// UpdateProductSuccess() : ..., ISuccess
// DisableProductSuccess() : ..., ISuccess

else if (notification is ISuccess)
    _viewModel.OK(new { });    // Data = objeto vacío {}
```

Usar cuando la acción no retorna datos (operaciones de escritura).

## Guía de DI — Lifetimes

### Singleton

Se crea **una sola instancia** para toda la vida de la aplicación. Todos los requests comparten la misma instancia.

**Cuándo usar:**
- Servicios sin estado mutable
- Cachés en memoria
- Clientes HTTP que deben reutilizarse
- Fábricas de conexión de base de datos

**En este proyecto:**
```csharp
services.AddSingleton<MainDbConnectionFactory>();
services.AddSingleton<IOpenDbConnectionFactory>(sp => sp.GetRequiredService<MainDbConnectionFactory>());
```

**⚠️ No inyectar servicios Scoped en un Singleton** — causará que el Scoped se comporte como Singleton (captive dependency).

---

### Scoped

Se crea **una instancia por request HTTP**. Todos los objetos creados en el mismo request comparten la misma instancia.

**Cuándo usar:**
- Todo lo que necesita vivir el tiempo de un request
- Conexiones de base de datos (una conexión por request)
- Clases Sql, repositorios, handlers, presenters, ViewModels

**En este proyecto:**
```csharp
services.AddScoped<MainDapperDbConnection>();   // una conexión por request
services.AddScoped<ProductsSql>();              // usa la conexión del request
services.AddScoped<IProductRepository, ProductRepository>();
services.AddScoped<INotificationHandler<GetProductResponse>, GetProductPresenter>();
services.AddScoped(typeof(ResultViewModel<>));  // shared entre controller y presenter del mismo request
```

**Por qué `ResultViewModel<>` es Scoped:** el controller y el presenter comparten la misma instancia dentro de un request. El presenter la llena; el controller la lee. Si fuera Singleton, habría race conditions entre requests.

---

### Transient

Se crea una **nueva instancia cada vez que se solicita**. No se comparte entre objetos.

**Cuándo usar:**
- Servicios ligeros sin estado que no necesitan ser compartidos
- Middlewares de HTTP message handlers

**En este proyecto:** no se usa Transient en el código de aplicación. Algunos middlewares internos de Common lo usan.

---

### Resumen de lifetimes del proyecto

| Tipo | Lifetime | Por qué |
|------|----------|---------|
| `MainDbConnectionFactory` | Singleton | Sin estado mutable, costoso de crear |
| `IOpenDbConnectionFactory` | Singleton | Alias del factory |
| `MainDapperDbConnection` | Scoped | Una conexión por request |
| `...Sql` classes | Scoped | Dependen de MainDapperDbConnection |
| `I...Repository` | Scoped | Dependen de ...Sql |
| `IRequestHandler<,>` | Scoped (auto) | AddMediator los registra automático |
| `INotificationHandler<>` (Presenters) | Scoped | Comparten ViewModel con el controller |
| `ResultViewModel<>` | Scoped | Compartido entre controller y presenter |

## Agregar un servicio externo (HTTP Client)

Si el módulo necesita llamar a una API externa:

```csharp
// Infrastructure/ServiceCollectionEx.cs
services.AddHttpClient<IMiServicioExterno, MiServicioExterno>(client =>
{
    client.BaseAddress = new Uri(configuration["MiServicio:BaseUrl"]!);
    client.DefaultRequestHeaders.Add("Accept", "application/json");
})
.AddCoreResilience();   // retry + circuit breaker de Common.Http
```

`AddCoreResilience()` viene de `Common.Http.HttpClientBuilderEx` y agrega políticas de resiliencia estándar.

## Checklist pre-build

Antes de hacer `dotnet build`, verificar:

### Capa Domain
- [ ] Entidad en `Domain/Entities/{Modulo}/` con `Id`, `PublicId`, campos de negocio, `CreatedAtUtc`, `UpdatedAtUtc`
- [ ] Interfaz de repositorio en `Domain/Repositories/{Modulo}/` con todos los métodos necesarios
- [ ] Propiedades `{ get; init; }` — sin setters públicos
- [ ] Sin referencias a Infrastructure, Application o WebApi

### Capa Infrastructure
- [ ] `{Entidad}Sql.cs` en `Infrastructure/Persistence/SQLDB/Main/{Modulo}/`
- [ ] Todos los métodos de la interfaz implementados en `ProductRepository.cs`
- [ ] `services.AddScoped<ProductsSql>()` en `Infrastructure/ServiceCollectionEx.cs`
- [ ] `services.AddScoped<IProductRepository, ProductRepository>()` en `Infrastructure/ServiceCollectionEx.cs`
- [ ] SQL como raw strings — sin concatenación ni interpolación
- [ ] Parámetros como `new { param }` — nunca strings con @param directo
- [ ] `INSERT` con `RETURNING` usa `QueryFirstAsync` con `!` null-forgiving

### Capa Application
- [ ] `{Accion}Request.cs` — `sealed record`, implementa `IRequest<{Accion}Response>`
- [ ] `Responses/{Accion}Response.cs` — `abstract record`, implementa `IResponse`
- [ ] `Responses/{Accion}Success.cs` — `sealed record`, implementa `ISuccess` o `ISuccess<TDto>`
- [ ] `Responses/{Accion}Failure.cs` — `sealed record`, implementa `INotFoundFailure` / `IConflictFailure` / etc.
- [ ] `{Accion}Handler.cs` — `sealed class`, implementa `IRequestHandler<{Accion}Request, {Accion}Response>`
- [ ] Handler no tiene lógica de UI ni referencias a WebApi o Infrastructure
- [ ] DTO en `Application/Dto/{Modulo}/` si el éxito retorna datos
- [ ] **No registrar el handler manualmente** — `AddMediator` lo descubre solo

### Capa WebApi
- [ ] `{Accion}Presenter.cs` en `WebApi/EndPoints/{Modulo}/Presenters/`
- [ ] Presenter implementa `IPresenter<{Accion}Response>` (o directamente `INotificationHandler<>`)
- [ ] Presenter registrado en `WebApi/ServiceCollectionEx.cs` como `INotificationHandler<{Accion}Response>`
- [ ] Controller extiende `BaseApiController`
- [ ] Controller inyecta `IMediator`, `ILogger<T>` y `ResultViewModel<{Modulo}Controller>`
- [ ] Todos los action methods usan `_ = await Mediator.Send(...)`
- [ ] Todos retornan `_viewModel.IsSuccess ? Ok(_viewModel) : StatusCode(500, _viewModel)`
- [ ] Todos tienen `catch` con unwrap de inner exception

### Capa Host (migraciones)
- [ ] Archivo `NNN_tabla.sql` en `Host/Services/Schema Migration/Tables/`
- [ ] `CREATE TABLE IF NOT EXISTS` — idempotente
- [ ] Archivo `NNN+1_tabla_indexes.sql` con índice único sobre `PublicId`
- [ ] Esquema `dbo` en todas las tablas
- [ ] Fechas: `TIMESTAMP(0) NOT NULL DEFAULT (timezone('utc', now()))`

### Build final
- [ ] `dotnet build back-template/Host/Host.csproj` — **0 errores, 0 warnings nuevos**
- [ ] `dotnet test back-template/Tests/Tests.csproj` — **0 tests fallidos**
